This page contains the working of this project

This project is a Flask application code that implements a simple inventory management system. The system includes functionalities for user registration, login, product management, sales history, generating bills, and generating balance reports using Plotly for data visualization. Here's an overview of the key components and functionalities of the code:

  1. Imports: The code starts with importing various libraries and modules required for the application, such as Flask, SQLAlchemy, pdfkit, and Plotly.
  2. Database Models: The code defines SQLAlchemy models for different tables in the database, including Product, Product_history, User, and SoldItem.
  3. Decorators and Helper Functions:
    • admin_required: A decorator that restricts access to certain routes to only users with the 'admin' user type.
    • unique_bill_id: A function to generate unique bill IDs for sold items.
    • update_product_history: A function to update the product history when a product is edited.
  4. Routes and Views:
    • /: Home page route that renders the "home.html" template.
    • /register: Handles user registration, hashing passwords, and adding users to the database.
    • /login: Handles user login and session management.
    • /my_website: Renders the "my_website.html" template.
    • /profile: Handles product addition and displays the user's profile page.
    • /dashboard: Displays statistics and information about products in the inventory.
    • /edit/<int:product_id>: Handles editing a product's details.
    • /delete/<int:product_id>: Handles deleting a product.
    • /products and /products_history: Display product and product history lists.
    • /cashier: Handles the selling of products to customers.
    • /sell_history: Displays a history of sold items.
    • /generate_bill/<string:bill_id>: Generates an HTML bill template for a sold item.
    • /pdf_bill/<string:bill_id>: Generates a PDF bill for a sold item.
    • /balance_report: Generates a balance report using Plotly for data visualization.
    • /logout: Logs the user out and redirects to the login page.
  5. App Initialization and Database Creation:
    • The application is initialized using the Flask app instance.
    • The SQLAlchemy database models are created and updated.
  6. Running the App:
    • The app is run in debug mode if the script is executed directly.
